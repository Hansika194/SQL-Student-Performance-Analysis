# SQL-Student-Performance-Analysis
SQL Data Analysis Internship –  Advanced SQL using JOIN, GROUP BY, HAVING, Subqueries, Student Performance Analysis.

## Project Title

**Student Performance Analysis Using SQLite**

## Objective

The objective of this project is to analyze student performance using a relational database with **Students**, **Courses**, and **Enrollments** tables. The project demonstrates how SQL can be used to retrieve, analyze, and summarize data using advanced SQL concepts such as JOINs, GROUP BY, HAVING, aggregate functions, and subqueries.

## Tools Used

* SQLite
* DB Browser for SQLite (or DB Fiddle)
* GitHub
* LinkedIn

## Database Structure

* **Students** – Stores student information.
* **Courses** – Stores course details.
* **Enrollments** – Links students with courses and stores grades.

## Queries Performed

* Displayed student course enrollment details using INNER JOIN.
* Identified the top-performing student in each course.
* Calculated the pass rate for each course.
* Found the overall top-performing student across all courses.
* Identified students enrolled in multiple courses.
* Used aggregate functions such as `COUNT()`, `AVG()`, and `MAX()`.
* Applied GROUP BY, HAVING, filtering, sorting, and subqueries to analyze data.

## Conclusion

This project provided hands-on experience in analyzing relational data using SQL. It strengthened my understanding of JOIN operations, aggregate functions, GROUP BY, HAVING, and subqueries while improving my ability to extract meaningful insights from a database. Through this task, I gained practical experience in solving real-world data analysis problems using SQL.
